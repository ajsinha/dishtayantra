Market-Aware Scheduling (v2.1.0)

Run DAGs only on chosen weekdays and never on market holidays - composed per DAG from one or more holiday calendars, with intraday edits honored automatically within five minutes.

Configuration

Add an optional schedule block to any DAG. All keys are optional; a DAG without one behaves exactly as before:

{ "name": "us_equities_session", "start_time": "0930", "duration": "6h30m", "schedule": { "days_of_week": ["mon", "tue", "wed", "thu", "fri"], "exclude_days_of_week": [], "holiday_calendars": ["USA", "CANADA"] } }

A DAG is active only when ALL of these hold:

  • The time window matches (wrap-around-aware - 2200-0600 works) or no window is configured.
  • days_of_week (allowlist): today is in the list. Accepts mon/tuesday/etc., case-insensitive.
  • exclude_days_of_week (denylist, applied after the allowlist): today is not in the list.
  • holiday_calendars: today is not a holiday in ANY listed calendar - union semantics, so a cross-listed strategy following USA and CANADA stands down whenever either market closes.
Holiday calendars

Calendars are plain JSON files under the storage prefix holidays.prefix (default config/holidays/):

{ "name": "USA", "description": "United States market holidays (NYSE-observed set)", "holidays": { "2026-01-01": "New Year's Day", "2026-11-26": "Thanksgiving Day" } }
  • Baked in: usa.json (NYSE set) and canada.json (TSX set), 150 holidays each covering 2026-2040, with observed-date rules and computus-derived Good Friday.
  • Regenerate or extend: python scripts/generate_holiday_calendars.py 2026 2055
  • Add your own: drop myexchange.json into the prefix and reference "MYEXCHANGE" - names are case-insensitive.
  • Fail fast: a DAG referencing a calendar that cannot be loaded refuses to load, with the exact file named in the error.
Intraday updates (< 5 minutes)

Both the calendar files and each DAG's schedule block can be edited while the server runs - no restart:

  • Calendar edits (e.g. declaring an ad-hoc market closure) reload on a TTL of holidays.reload_seconds (default 180, hard-capped at 300).
  • DAG schedule edits are re-read from storage every schedule.refresh_seconds (default 240, hard-capped at 240) and applied to the live DAG with a boxed log entry.
  • Combined with the 60-second schedule monitor, the worst-case reaction time is under five minutes.
  • A broken edit is rejected - the error is logged with a full stack trace and the last good rules stay in force, so a typo never flips a trading DAG on.
DAGs suspended or resumed by the scheduler log boxed DAG SCHEDULE: AUTO-SUSPEND / AUTO-RESUME entries including the reason (e.g. holiday - USA: Good Friday); the dashboard and DAG details show the schedule and its current evaluation.
