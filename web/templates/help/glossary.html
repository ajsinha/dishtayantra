Glossary of Terms

Definitions of key terms and concepts used throughout DishtaYantra.

A B C D E F G I J K M N P Q R S T U W
A
ActiveMQ

Apache ActiveMQ is an open-source message broker that implements JMS (Java Message Service). DishtaYantra supports both queue and topic messaging with ActiveMQ using the activemq:// protocol.

AutoClone

A feature that automatically creates multiple copies (clones) of a DAG during specified time periods to handle peak loads. Configured using ramp_up_time, duration, and ramp_count properties.

B
Bootstrap Servers

A list of Kafka broker addresses used by Kafka clients to establish initial connection to the Kafka cluster. Specified as host:port pairs.

C
Cache

An in-memory key-value store in DishtaYantra that provides fast data access. It's Redis-compatible and supports TTL (time-to-live), persistence, and pattern-based queries.

Calculator

A processing component used in CalculationNodes to perform data transformations, calculations, filtering, or enrichment. Examples include AdditionCalculator, ApplyDefaultsCalculator, and PassthruCalculator.

CalculationNode

A DAG node type that processes data using a Calculator. It receives data from upstream nodes, applies the calculator's logic, and passes results downstream.

Clone

A copy of a DAG created either manually or automatically (via AutoClone). Clones run independently with their own state but share the same configuration structure.

Consumer Group

In Kafka, a group of consumers that work together to consume messages from topics. Messages are distributed among group members, enabling parallel processing and load balancing.

D
DAG (Directed Acyclic Graph)

The core processing unit in DishtaYantra. A DAG consists of nodes connected by edges, where data flows in one direction without cycles. DAGs define data pipelines for processing, transformation, and routing.

DAG Designer

A visual drag-and-drop interface for creating and editing DAG configurations without writing JSON code manually.

DAG Server

The core engine that manages DAG lifecycle including loading, starting, stopping, and monitoring DAGs. It also handles time window checking and AutoClone operations.

Data Transformer

See Transformer.

Destination

The target location where a Publisher sends data. Specified as a URI like kafka://topic/events or mem://queue/output.

Duration

The length of time a DAG should run within its time window. Specified in formats like 8h (8 hours), 30m (30 minutes), or 1h30m (1 hour 30 minutes).

E
Edge

A connection between two nodes in a DAG that defines data flow direction. Edges can optionally include a transformer to modify data as it passes through.

F
Fan-in

A pattern where multiple data sources are combined into a single stream. In DishtaYantra, use composite:// or fanin:// source to merge multiple subscribers.

Fan-out

A pattern where data from a single source is distributed to multiple destinations. Achieved by adding multiple publishers to a PublicationNode.

Free-Threading

A Python build option (available in Python 3.13+) that removes the Global Interpreter Lock (GIL), enabling true parallel execution of Python threads across multiple CPU cores. Build Python with --disable-gil to enable. See Free-Threading Guide.

G
GIL (Global Interpreter Lock)

A mutex in standard Python that prevents multiple threads from executing Python bytecode simultaneously. Limits parallelism to one thread at a time. Can be disabled in Python 3.13+ with free-threading builds.

gRPC

A high-performance RPC (Remote Procedure Call) framework. DishtaYantra supports gRPC-based pub/sub using the grpc:// protocol.

I
In-Memory Queue

A queue stored entirely in memory for fast message passing. Accessed using mem://queue/name. Data is lost on server restart unless persisted.

In-Memory Topic

A publish-subscribe topic stored in memory. Multiple subscribers can receive the same message. Accessed using mem://topic/name.

J
JSON Configuration

The format used to define DAG structure, including nodes, edges, subscribers, publishers, calculators, and transformers. Can be written manually or generated by the DAG Designer.

JSONL

JSON Lines format where each line is a separate JSON object. Commonly used for file-based data streams with the file:// protocol.

K
Kafka

Apache Kafka is a distributed event streaming platform. DishtaYantra supports Kafka topics using the kafka://topic/ protocol for high-throughput data processing.

M
Max Depth

The maximum number of messages a subscriber's internal buffer can hold. When reached, older messages may be dropped or the subscriber may block.

Metronome

A special subscriber/publisher type that generates periodic events at specified intervals. Useful for heartbeats, scheduled tasks, and time-based triggers.

N
Node

A vertex in a DAG that performs a specific function. Types include SubscriptionNode (input), CalculationNode (processing), PublicationNode (output), MetronomeNode (timer), and SinkNode (discard).

P
Perpetual DAG

A DAG without a time window that runs continuously until manually stopped. Created by omitting start_time and duration from the configuration.

Publisher

A component that sends data to an external destination (queue, topic, file, API, etc.). Publishers are attached to PublicationNodes.

PublicationNode

A DAG node type that outputs data to one or more publishers. Typically the final node(s) in a DAG where processed data is sent to external systems.

Py4J

A bridge library enabling Python programs to call Java code. DishtaYantra uses Py4J to invoke high-performance Java calculators with gateway pooling for parallel execution. See Py4J Integration Guide.

pybind11

A lightweight header-only C++ library that exposes C++ types to Python with minimal overhead (~100ns per call). DishtaYantra uses pybind11 for native C++ calculators with SIMD optimization and zero-copy memory sharing. See pybind11 Integration Guide.

PyO3

Rust bindings for Python, allowing Rust code to be called from Python as a native extension. DishtaYantra uses PyO3 for memory-safe, thread-safe high-performance calculators. See Rust Integration Guide.

Q
Queue

A point-to-point messaging pattern where each message is consumed by exactly one consumer. Contrast with Topic (publish-subscribe).

R
RabbitMQ

An open-source message broker supporting AMQP protocol. DishtaYantra supports both queues and topics using rabbitmq://queue/ and rabbitmq://topic/.

Ramp Count

In AutoClone configuration, the number of DAG clones to create during the ramp-up phase.

Ramp Up Time

In AutoClone configuration, the time (HHMM format) when clone creation begins.

Rust

A systems programming language known for memory safety without garbage collection. DishtaYantra supports Rust calculators via PyO3, providing compile-time memory safety and fearless concurrency. See Rust Integration Guide.

REST Calculator

A calculator type that invokes external HTTP/REST endpoints via POST requests. Supports API Key, Basic Auth, and Bearer token authentication. Useful for integrating microservices and third-party APIs. See REST Integration Guide.

Redis

An in-memory data structure store. DishtaYantra supports Redis keys (redis://) and pub/sub channels (redischannel://).

Role

A permission level assigned to users. DishtaYantra supports three roles: admin (full access), operator (start/stop DAGs), and user (view only).

S
SinkNode

A DAG node type that consumes data without producing output. Useful for logging, metrics collection, or discarding unwanted data branches.

Source

The origin location from which a Subscriber reads data. Specified as a URI like kafka://topic/events or mem://queue/input.

Start Time

The time when a DAG's time window begins, specified in HHMM format (e.g., 0900 for 9:00 AM).

Subscriber

A component that reads data from an external source (queue, topic, file, API, etc.). Subscribers are attached to SubscriptionNodes.

SubscriptionNode

A DAG node type that ingests data from a subscriber. Typically the entry point(s) of a DAG where external data enters the pipeline.

System Monitoring

Admin-only dashboard showing real-time system metrics including CPU, memory, disk, network statistics, DAG server health, and calculator integration status. Accessible via Admin â†’ System Monitoring. Added in version 1.1.1.

T
TIBCO EMS

TIBCO Enterprise Message Service, an enterprise messaging platform. Supported via tibcoems://queue/ and tibcoems://topic/.

Time Window

A scheduled period during which a DAG is active. Defined by start_time and duration. Outside this window, the DAG is suspended.

Topic

A publish-subscribe messaging pattern where messages are broadcast to all subscribers. Contrast with Queue (point-to-point).

Transformer

A component that modifies data as it flows through edges or at node boundaries. Used for filtering attributes, applying defaults, or reshaping data structure.

TTL (Time To Live)

The duration in seconds that a cache entry remains valid before automatic expiration and deletion.

U
URI (Uniform Resource Identifier)

The format used to specify sources and destinations in DishtaYantra. Format: protocol://type/name (e.g., kafka://topic/events).

W
WebSphere MQ (IBM MQ)

IBM's enterprise messaging middleware. Supported via websphere://queue/ and websphere://topic/.
