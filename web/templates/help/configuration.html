Configuration File Overview

DishtaYantra uses several configuration files located in the config/ directory:

File Purpose Format
application.properties Server, Flask, LMDB settings Properties (key=value)
worker_config.json Worker pool, affinity, monitoring JSON
users.json User accounts and roles JSON
dags/*.json DAG definitions JSON
application.properties Properties Format

Main application configuration for server, Flask, and LMDB settings.

Location: config/application.properties

# MCP Server Configuration
app.name=DishtaYantra
app.version=1.5.2

server.host=0.0.0.0
server.port=5002
server.debug=False

# Flask Configuration
flask.secret_key=${FLASK_SECRET_KEY:default_secret_key}

# External Module Paths
external.module.path.1=/opt/custom_modules
external.module.path.2=${HOME}/my_plugins

# User Registry Configuration
user.registry.reload_interval=600

# LMDB Zero-Copy Data Exchange
lmdb.db.path=${LMDB_DB_PATH:/tmp/dishtayantra_lmdb}
lmdb.map.size=1073741824
lmdb.max.dbs=100
lmdb.ttl.seconds=300
lmdb.max.readers=126
lmdb.cleanup.interval=60
Properties Reference
Property Default Description
app.name DishtaYantra Application display name
server.host 0.0.0.0 Server bind address
server.port 5002 Server port number
flask.secret_key - Flask session secret (change in production!)
user.registry.reload_interval 600 Seconds between users.json reloads
lmdb.db.path /tmp/dishtayantra_lmdb LMDB database path for calculator data exchange
lmdb.map.size 1073741824 LMDB max size in bytes (1GB)
Environment Variables: Use ${VAR_NAME:default} syntax for environment variable substitution.
worker_config.json v1.5.2

Configuration for the multiprocessing worker pool, DAG affinity, and cross-worker communication.

Location: config/worker_config.json

{
    "worker_pool": {
        "enabled": false,  // Set to true to enable
        "num_workers": 4,
        "min_workers": 2,
        "max_workers": 32,
        "worker_startup_timeout_seconds": 30,
        "worker_shutdown_timeout_seconds": 10,
        "health_check_interval_seconds": 5,
        "status_report_interval_seconds": 2,
        "auto_restart_on_crash": true,
        "max_restart_attempts": 3,
        "restart_backoff_seconds": 5
    },
    
    "affinity": {
        "default_strategy": "weight_based",
        "rebalance_enabled": true,
        "rebalance_threshold": 0.3,
        "rebalance_interval_seconds": 60,
        "allow_dag_pinning": true,
        "allow_exclusive_workers": true
    },
    
    "communication": {
        "control_queue_maxsize": 1000,
        "status_queue_maxsize": 5000,
        "use_lmdb_for_cross_worker": true,
        "lmdb_path": "data/worker_lmdb",
        "lmdb_map_size_mb": 1024
    },
    
    "monitoring": {
        "collect_cpu_metrics": true,
        "collect_memory_metrics": true,
        "collect_dag_metrics": true,
        "metrics_history_size": 1000
    },
    
    "defaults": {
        "dag_cpu_weight": 0.2,
        "dag_priority": 5
    }
}
Worker Pool Section
Option Type Default Description
enabled boolean true Enable/disable worker pool
num_workers int/string 4 Number of workers. Use "auto" for CPU count.
min_workers int 2 Minimum workers (used with "auto")
max_workers int 32 Maximum workers (used with "auto")
worker_startup_timeout_seconds int 30 Timeout waiting for worker to start
worker_shutdown_timeout_seconds int 10 Timeout waiting for worker to stop
health_check_interval_seconds int 5 Health check frequency
status_report_interval_seconds int 2 How often workers report status
auto_restart_on_crash boolean true Auto-restart crashed workers
max_restart_attempts int 3 Max consecutive restart attempts
restart_backoff_seconds int 5 Initial backoff before restart (exponential)
Affinity Section
Option Type Default Description
default_strategy string "weight_based" DAG assignment strategy: round_robin, weight_based, least_loaded, random
rebalance_enabled boolean true Enable automatic load rebalancing
rebalance_threshold float 0.3 Load difference (0-1) to trigger rebalance
rebalance_interval_seconds int 60 How often to check for rebalancing
allow_dag_pinning boolean true Allow DAGs to pin to specific workers
allow_exclusive_workers boolean true Allow DAGs to request exclusive workers
Communication Section
Option Type Default Description
control_queue_maxsize int 1000 Max control messages per worker queue
status_queue_maxsize int 5000 Max status messages in shared queue
use_lmdb_for_cross_worker boolean true Enable LMDB-based cross-worker pub/sub
lmdb_path string "data/worker_lmdb" LMDB database path for worker pub/sub
lmdb_map_size_mb int 1024 LMDB max size in megabytes
Monitoring Section
Option Type Default Description
collect_cpu_metrics boolean true Collect CPU usage per worker
collect_memory_metrics boolean true Collect memory usage per worker
collect_dag_metrics boolean true Collect per-DAG execution metrics
metrics_history_size int 1000 Number of metric samples to retain
users.json JSON Format

User accounts, roles, and permissions.

Location: config/users.json

{
    "users": [
        {
            "username": "admin",
            "password_hash": "...",
            "role": "admin",
            "email": "admin@example.com",
            "active": true
        }
    ]
}
User Roles
Role Permissions
admin Full access: manage users, DAGs, system settings
operator Start/stop DAGs, view status, publish messages
user View DAGs and status (read-only)
DAG Configuration Files JSON Format

DAG definitions with nodes, connections, and worker affinity.

Location: config/dags/*.json

{
    "name": "my_dag",
    "description": "Example DAG",
    
    // Worker Affinity (v1.5.2)
    "worker_affinity": {
        "pinned_worker": 0,        // Pin to specific worker
        "exclusive": false,        // Request dedicated worker
        "preferred_workers": [0, 1]  // Preference list
    },
    
    "nodes": [
        {
            "name": "subscriber",
            "type": "subscriber",
            "config": { ... }
        }
    ],
    
    "edges": [
        ["node1", "node2"]
    ]
}
See DAG Configuration Guide for complete documentation.
