Calculators

Data processing components that perform calculations, filtering, and transformations on data.

Overview

Calculators are used in CalculationNode to process data. They receive a data dictionary and return a processed result.

{
  "name": "my_calculator",
  "type": "ApplyDefaultsCalculator",
  "config": {
    "defaults": {"status": "active"}
  }
}
NullCalculator

Returns a deep copy of the input data. Useful for creating isolated copies.

{
  "name": "copy_calc",
  "type": "NullCalculator",
  "config": {}
}

// Input:  {"id": 1, "name": "test"}
// Output: {"id": 1, "name": "test"}  (deep copy)
PassthruCalculator

Returns the input data as-is (same reference). Most efficient for pass-through scenarios.

{
  "name": "passthru_calc",
  "type": "PassthruCalculator",
  "config": {}
}
ApplyDefaultsCalculator

Applies default values for missing or null attributes.

{
  "name": "defaults_calc",
  "type": "ApplyDefaultsCalculator",
  "config": {
    "defaults": {
      "status": "pending",
      "priority": 1,
      "processed": false
    }
  }
}

// Input:  {"id": 1, "status": null}
// Output: {"id": 1, "status": "pending", "priority": 1, "processed": false}
AttributeFilterCalculator

Keeps only the specified attributes, removing all others.

{
  "name": "filter_calc",
  "type": "AttributeFilterCalculator",
  "config": {
    "keep_attributes": ["id", "name", "timestamp"]
  }
}

// Input:  {"id": 1, "name": "test", "secret": "xxx", "timestamp": 123}
// Output: {"id": 1, "name": "test", "timestamp": 123}
AttributeFilterAwayCalculator

Removes the specified attributes, keeping all others.

{
  "name": "remove_secrets_calc",
  "type": "AttributeFilterAwayCalculator",
  "config": {
    "filter_attributes": ["password", "secret", "token"]
  }
}

// Input:  {"id": 1, "name": "test", "password": "xxx", "data": "value"}
// Output: {"id": 1, "name": "test", "data": "value"}
AdditionCalculator

Adds the values of specified attributes and outputs the sum.

{
  "name": "sum_calc",
  "type": "AdditionCalculator",
  "config": {
    "arguments": ["price", "tax", "shipping"],
    "output_attribute": "total"
  }
}

// Input:  {"price": 100, "tax": 10, "shipping": 5}
// Output: {"total": 115}
MultiplicationCalculator

Multiplies the values of specified attributes.

{
  "name": "multiply_calc",
  "type": "MultiplicationCalculator",
  "config": {
    "arguments": ["quantity", "unit_price"],
    "output_attribute": "line_total"
  }
}

// Input:  {"quantity": 5, "unit_price": 20}
// Output: {"line_total": 100}
AttributeNameChangeCalculator

Renames attributes based on a mapping.

{
  "name": "rename_calc",
  "type": "AttributeNameChangeCalculator",
  "config": {
    "name_mapping": {
      "old_id": "new_id",
      "user_name": "username",
      "dt": "timestamp"
    }
  }
}

// Input:  {"old_id": 1, "user_name": "john", "dt": 123}
// Output: {"new_id": 1, "username": "john", "timestamp": 123}
Custom Calculators

Create your own calculators by extending DataCalculator:

from core.calculator.core_calculator import DataCalculator

class MyCustomCalculator(DataCalculator):
    def calculate(self, data):
        self._calculation_count += 1
        # Your custom logic here
        result = data.copy()
        result['processed_by'] = 'MyCustomCalculator'
        return result

Reference custom calculators using the full module path:

{
  "name": "custom_calc",
  "type": "mymodule.calculators.MyCustomCalculator",
  "config": {}
}
Arrow Columnar Calculators (opt-in, new style)

An ArrowCalculator processes a columnar batch of records with vectorized pyarrow.compute kernels instead of one dict at a time. It subclasses DataCalculator, so it is a drop-in for a row calculator and the engine is unchanged.

Old-style (row) and new-style (Arrow) calculators coexist — in the same running instance and even within the same graph. On single-dict flow an Arrow calculator is an exact drop-in (output-identical to the row version). On batched flow it vectorizes for the speedup, and any legacy row stage on that path is wrapped with RowCalculatorBatchAdapter.

from core.calculator.arrow_calculator import ArrowCalculator, append_columns
import pyarrow.compute as pc

class MyArrowCalculator(ArrowCalculator):
    def calculate_batch(self, batch):
        doubled = pc.multiply(batch.column("value"), 2)
        return append_columns(batch, {"doubled": doubled})

Use it like any custom calculator (works in a graph alongside row calculators):

{
  "name": "my_arrow",
  "type": "mymodule.MyArrowCalculator",
  "config": {}
}

When it helps: high-volume, numeric/columnar, schema-regular, latency-tolerant streams (and native C++/Rust/Java calculators via zero-copy). When it does not: low-volume, ultra-low-latency per-event, branchy/string-heavy, or I/O-bound paths. See docs/design/A1-worked-example-and-coexistence.md and run python -m perftest.run_arrow_example.
