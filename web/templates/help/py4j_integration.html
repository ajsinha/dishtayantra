Architecture Overview

DishtaYantra v1.6.0 introduces centralized JVM management for Java calculator integration via Py4J.

v1.6.0 Architecture
DishtaYantra → JVM Manager → Py4J Gateway Pool → Java JVM(s)
Config-based initialization • Multiple JVM support • Auto-restart • Health monitoring
Key Components
Component Description
JVM Manager NEW Central manager for JVM instances and gateway connections. Handles config loading, lifecycle management, and health monitoring.
jvm_config.json Configuration file defining gateways, JVM settings, and pre-configured calculators.
Gateway Pool Pool of Py4J gateway connections for concurrent access to Java calculators.
Java Calculators High-performance calculators implementing the Calculator interface.
JVM Manager NEW in 1.6.0

The JVM Manager provides centralized management of JVM instances and Py4J gateway connections.

Features
  • Config-based initialization - Define everything in jvm_config.json
  • Auto-start on load - JVMs start automatically when application properties are loaded
  • Multiple gateway support - Configure primary, secondary, and custom gateways
  • Health monitoring - Automatic health checks and reconnection
  • Worker process awareness - Both main and worker processes can access gateways
Python API
from core.jvm import get_jvm_manager, get_gateway, create_calculator # Get the JVM Manager singleton jvm_manager = get_jvm_manager() # Check if initialized if jvm_manager.is_initialized(): # Get a gateway connection gateway = get_gateway("primary") # Create a calculator calc = create_calculator("MathCalculator", "my_calc", {"operation": "sum"}) # Get status status = jvm_manager.get_status() print(status)
Configuration (jvm_config.json)

JVM configuration is stored in config/jvm_config.json:

{ "jvm_manager": { "enabled": true, "auto_start_on_load": true, "health_check_interval_seconds": 30 }, "gateways": [ { "name": "primary", "enabled": true, "host": "localhost", "base_port": 25333, "pool_size": 4, "jvm": { "auto_start": true, "heap_size_mb": 512, "max_heap_size_mb": 2048, "jvm_options": ["-XX:+UseG1GC"], "classpath": ["java/lib/*"] }, "connection": { "timeout_seconds": 30, "retry_attempts": 3 } } ], "calculators": { "definitions": { "MathCalculator": { "java_class": "com.dishtayantra.calculators.examples.ExampleCalculators$MathCalculator", "description": "Mathematical operations", "gateway": "primary", "default_config": {"operation": "sum"} } } } }
Configuration Options
Section Option Description
jvm_manager enabled Enable/disable the JVM Manager
auto_start_on_load Start JVMs when application loads
gateways base_port Starting port for gateway pool (e.g., 25333-25336)
pool_size Number of gateway connections
jvm.heap_size_mb Initial JVM heap size
jvm.classpath Java classpath for calculator JARs

Using Java Calculators in DAGs

To use a Java calculator in a DAG, add an entry to the calculators section with "type": "java" and the fully-qualified java_class in its config. The builder routes it through CalculatorFactory to the Py4J gateway. Nodes then reference it by name.

{ "name": "calculate_pricing", "type": "java", "config": { "java_class": "com.dishtayantra.calculators.examples.ExampleCalculators$TradePricingCalculator", "gateway_config": { "gateway": "primary" }, "commission_rate": 0.001, "tax_rate": 0.002, "include_vat": true } }
Tip: If the calculator is defined in jvm_config.json, you can reference it by name and the java_class will be resolved automatically.

Writing Custom Calculators

Create custom Java calculators by extending AbstractCalculator:

package com.yourcompany.calculators; import com.dishtayantra.calculators.AbstractCalculator; import java.util.Map; import java.util.HashMap; public class MyCustomCalculator extends AbstractCalculator { private final double factor; public MyCustomCalculator(String name, Map<String, Object> config) { super(name, config); this.factor = getConfig("factor", 1.0); } @Override protected Map<String, Object> doCalculate(Map<String, Object> data) { Map<String, Object> result = new HashMap<>(data); // Your calculation logic here double value = ((Number) data.get("value")).doubleValue(); result.put("result", value * factor); return result; } }
Building and Deploying
# Compile your calculator javac -cp "dishtayantra-calculators.jar:py4j.jar" MyCustomCalculator.java # Create a JAR jar cvf my-calculators.jar com/yourcompany/calculators/*.class # Add to classpath in jvm_config.json "classpath": ["java/lib/*", "path/to/my-calculators.jar"]

Troubleshooting

  1. Install Py4J: pip install py4j
  2. Check config/jvm_config.json exists and enabled: true
  3. Ensure Java is installed: java -version
  4. Check server logs for errors

  1. Check if the Java gateway server is running on the expected port
  2. Verify port is not blocked by firewall
  3. Try reconnecting via /api/jvm/gateways/primary/reconnect
  4. Check JVM process PID in JVM Management page

  1. Verify the JAR file is in the classpath defined in jvm_config.json
  2. Check the fully qualified class name is correct
  3. For inner classes, use $ notation: OuterClass$InnerClass
  4. Restart the JVM gateway server after adding new JARs
