What is DishtaYantra?

DishtaYantra (Sanskrit: दिष्टयन्त्र - "Directed Instrument") is a powerful, enterprise-grade Directed Acyclic Graph (DAG) execution engine designed for real-time data processing and workflow orchestration.

Built with Python and Flask, DishtaYantra provides a flexible framework for creating complex data pipelines that can:

  • Ingest data from multiple sources including Kafka, RabbitMQ, ActiveMQ, TIBCO EMS, WebSphere MQ, REST APIs, Files, and more
  • Process and transform data using customizable calculators and transformers
  • Publish results to various destinations in real-time
  • Scale horizontally using the AutoClone feature for high-throughput scenarios
  • Schedule execution using time windows for controlled processing periods

Core Architecture Components
  • WorkerPoolManager - Orchestrates worker lifecycle, DAG assignments, and health monitoring
  • DAGWorkerProcess - Individual worker process that owns multiple DAGs, shares calculator registry
  • DAGAffinityManager - Handles DAG-to-worker assignment with load balancing strategies
  • WorkerHealthMonitor - Tracks health via heartbeats, triggers auto-restart on failure
Worker Management Features
  • DAG Migration - Move DAGs between workers at runtime
  • Exclusive Workers - Dedicate a worker to a single critical DAG
  • Load Rebalancing - Automatic rebalancing when imbalance exceeds threshold
  • Manual Restart - Restart individual workers via API or UI
DAG Affinity Strategies
Strategy Description
weight_basedDefault - Assigns DAGs to worker with lowest estimated CPU weight
round_robinDistributes DAGs evenly across workers in rotation
least_loadedUses actual CPU metrics for assignment decisions
randomRandom assignment (useful for testing)
Health Monitoring
  • Heartbeat interval: 2 seconds
  • Unhealthy threshold: 3 missed heartbeats (15s)
  • Restart backoff: 5s → 10s → 20s (max 60s)
  • Max restart attempts: 3 (configurable)
Configuration
  • Config file: config/worker_config.json
  • Workers: 2-32 (or "auto" for CPU count)
  • Cross-worker: LMDB Pub/Sub
  • API: /api/workers/* endpoints
