Logging Control

Change log levels at runtime — no restart needed. Lowering a chatty logger (for example the per-message subscriber/publisher loggers) to WARNING is the quickest way to cut logging overhead under load. Changes are temporary: everything reverts to the level in application.properties (logging.level) on the next restart. {% if worker_pool_running %} Changes are also broadcast to the {{ worker_count }} worker process(es). {% else %} The worker pool is not running, so changes apply to this process only. {% endif %}

Global (root) log level

Applies to every logger that doesn't have its own override below. Current: {{ root_level }}

DEBUG = everything (very verbose; turns per-message tracing back on). INFO = normal. WARNING/ERROR = quieter; good for cutting overhead. CRITICAL = effectively silent (you'll still see only the most severe events — not recommended for long periods).

Per-logger overrides

Target a single subsystem without changing everything else. The first few are the hot-path loggers — setting them to WARNING silences per-message logging while leaving the rest at the global level. Effective is the level actually in force (an override, or inherited from the root). Choose INHERIT to remove an override.
