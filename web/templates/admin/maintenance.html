Maintenance / Drain

Drain mode freezes subscribers so they stop pulling new messages from their brokers, while everything already in flight keeps flowing — so queues and publications drain. This is different from pause, which stops the whole flow and holds the queues. Use it before a maintenance window: freeze → watch everything drain to zero → then restart. Works for every broker (Kafka also gets a broker-level pause to avoid a consumer-group rebalance). Freeze state is temporary and clears on restart. {% if not worker_mode %}
